# Procurement Tracking App

An operational dashboard for managing procurement requests, vendors, purchase orders, and receiving, built with a modern React + Node + GraphQL + Prisma stack.

## Architecture
- **Frontend**: React, TypeScript, Apollo Client, Vite, plain CSS (Foundry/Workshop-inspired design)
- **Backend**: Node.js, Express, Apollo Server, GraphQL, Prisma ORM
- **Database**: PostgreSQL
- **Auth**: JWT-based via HTTP-only cookies, BCrypt for password hashing

## Prerequisites
- Node.js (v18+)
- PostgreSQL database

## Local Development

1. **Install dependencies:**
   Navigate into both directories to install dependencies.
   ```bash
   cd server && npm install
   cd ../client && npm install
   cd ..
   ```

2. **Environment Variables:**
   Create `server/.env` with:
   ```env
   DATABASE_URL="postgresql://user:password@localhost:5432/procurement_db"
   PORT=4000
   CLIENT_URL="http://localhost:5173"
   JWT_SECRET="replace_this_with_a_secure_secret_in_production"
   JWT_EXPIRES_IN="7d"
   NODE_ENV="development"
   ```
   Create `client/.env` with:
   ```env
   VITE_GRAPHQL_URL="http://localhost:4000/graphql"
   ```

3. **Initialize Database:**
   ```bash
   cd server
   npm run prisma:migrate
   npm run prisma:seed
   ```

4. **Run Development Servers:**
   From the root of the repository, you can start both servers:
   ```bash
   npm run dev
   ```

## Deployment Guide

### A. Neon Database Setup
- Create a PostgreSQL database on Neon.
- Copy your connection strings.
- Set `DATABASE_URL` to the pooled connection string (used for app runtime).
- Set `DIRECT_URL` to the direct connection string (used for Prisma migrations).

### B. Render Backend Setup
- Create a new Web Service from your GitHub repo in Render.
- **Root directory**: `server`
- **Build command**: `npm install && npm run build && npx prisma generate`
- **Start command**: `npm start`
- **Environment variables**:
  - `DATABASE_URL` = (your Neon connection string)
  - `CLIENT_URL` = (temporary frontend URL, update after Vercel deploy)
  - `JWT_SECRET` = (a long, secure random string)
  - `JWT_EXPIRES_IN` = `7d`
  - `NODE_ENV` = `production`

### C. Run Migrations
Against the production database, you will need to apply your schema and seed data.
From your local machine (with your `.env` temporarily pointing to the Neon DB) or via a Render deployment script, run:
```bash
npx prisma migrate deploy
npm run prisma:seed
```

### D. Vercel Frontend Setup
- Import your GitHub repo into Vercel.
- **Root directory**: `client`
- **Build command**: `npm run build`
- **Output directory**: `dist`
- **Environment variable**:
  - `VITE_GRAPHQL_URL` = `https://YOUR_RENDER_BACKEND_URL/graphql`

### E. After Frontend Deploy
- Copy the live production URL generated by Vercel (e.g., `https://procurement-app.vercel.app`).
- Go back to your Render backend dashboard.
- Update the `CLIENT_URL` environment variable to match the Vercel URL exactly (no trailing slash).
- Redeploy the backend so the CORS policy properly accepts the new frontend origin.

### F. Palantir Note
- The Foundry Developer Console supports static frontend hosting, but not Node/Express backend hosting.
- The current application can be deployed normally via Vercel/Render first to test functionality.
- A Foundry-native version would require rewriting the backend to integrate with OSDK/Objects/Workshop at a later stage.
